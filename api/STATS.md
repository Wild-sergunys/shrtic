# Сбор статистики переходов SHRTIC

## Обзор

При каждом переходе по короткой ссылке автоматически собирается статистика. Данные сохраняются в PostgreSQL.

## Что собирается

| Параметр | Источник | Пример |
|----------|----------|--------|
| Браузер | User-Agent | Chrome, Firefox, Safari, Other |
| Устройство | User-Agent | Desktop, Mobile, Tablet |
| Страна | IP-адрес → ip-api.com | Россия, США, Германия |
| Источник | Referer | Прямой, Twitter, Telegram, Google |

## Как работает

1. При редиректе (GET /r/{code}) хендлер асинхронно запускает сбор статистики
2. Парсится User-Agent: определение браузера и устройства
3. Определяется IP клиента (X-Forwarded-For → X-Real-IP → RemoteAddr)
4. IP отправляется в ip-api.com для определения страны
5. Заголовок Referer парсится для определения источника перехода
6. Все данные сохраняются в таблицу stats

## Агрегация

При запросе GET /api/links/{id}/stats статистика агрегируется через SQL GROUP BY:
- Группировка по браузеру, устройству, стране, источнику
- Подсчёт количества и процентов для каждого значения

## Ограничения

- На localhost страна всегда "Unknown"
- User-Agent от curl определяется как "Other"
- Для точного определения страны нужен внешний IP (VPS или ngrok)
